## What it gives your agent

- **A router + capsules** — one Markdown file per domain; the agent loads only what each session needs, with no cross-domain bleed.
- **Persistent memory** — a portable protocol for the durable facts your agent should recall every session. See [Memory](/polydomain/guides/memory/).
- **An optional notes/Obsidian knowledge base** — an organized, navigable store the agent reads and writes for near-infinite context. See [Connect a notes store](/polydomain/guides/notes-store/).
- **Lossless adoption** — already have a repo? [`adopt`](/polydomain/guides/adopt/) brings it up to the standard on a git safety branch, losslessly.
- **Actions on macOS** — let the agent do reminders, calendar events, and email via verified recipes, under a safety gate. See [Actions (macOS)](/polydomain/guides/actions/). _(macOS-first; the rest is cross-platform.)_
- **A safety gate** — `plan → review → approve → execute`. No runtime, no telemetry: it's an operating manual your agent reads.

```bash
npx polydomain init my-center --references obsidian
```
